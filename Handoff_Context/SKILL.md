# Handoff Context Skill

## 功能说明

该技能用于将当前 Agent 会话的上下文压缩成一个 Markdown 文件（handoff_context.md），以便下一个 Agent 可以立即接手工作。

## 使用场景

当需要切换到新的 Agent 会话时，调用此技能来生成上下文交接文件。

## 调用时机

在以下情况下应该调用此技能：
- 用户明确要求切换到新的 Agent 会话
- 当前会话达到复杂度上限，需要分批处理
- 需要将当前工作状态传递给另一个 Agent

## 输出内容

生成的 `handoff_context.md` 文件将严格包含以下内容：

### 当前目标

清晰描述当前要解决的核心问题。

### 研究结论

- 列出已确认相关的具体文件路径
- 指出关键的代码位置（行号）和函数名
- 简述对当前系统/代码逻辑的理解（只保留事实）

### 执行计划

- 详细列出接下来的具体实施步骤
- 每一步需要修改哪个文件？做什么改动？
- 每一步改动后如何验证/测试？

## 输出规范

- 文件名：`handoff_context.md`
- 文件位置：当前工作目录
- 格式：Markdown
- 语言：与当前对话语言保持一致

## 注意事项

- 只保留对下一个 Agent 有用的"高信噪比"信息
- 剔除所有无关的对话历史和尝试过程
- 不要包含之前的聊天记录、报错日志或失败的路径
- 只保留事实性的信息，不要保留搜索过程
- 保持内容简洁、清晰、可操作
